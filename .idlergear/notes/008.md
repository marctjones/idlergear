# Research: Existing MCP Filesystem Servers

## Executive Summary

**RECOMMENDATION: DO NOT BUILD** - Multiple high-quality MCP filesystem servers already exist with comprehensive features. Instead, INTEGRATE the official @modelcontextprotocol/server-filesystem.

## Official Anthropic Filesystem MCP Server

**Package:** `@modelcontextprotocol/server-filesystem` (Node.js)
**Status:** Official reference implementation, actively maintained
**Published:** December 2025 (very recent)

### Tools Provided (12 total)

#### Read Operations
1. **read_text_file** - Read complete file contents as text, with optional head/tail parameters
2. **read_media_file** - Access images/audio, returns base64-encoded data with MIME type
3. **read_multiple_files** - Read multiple files simultaneously with partial failure tolerance

#### Directory Operations
4. **list_directory** - Display directory contents with [FILE]/[DIR] prefixes
5. **list_directory_with_sizes** - Show entries with file sizes and sorting options
6. **directory_tree** - Generate recursive JSON tree structure (THIS IS THE KEY FEATURE)
7. **create_directory** - Create new directories or verify existing ones

#### File Manipulation
8. **write_file** - Create or overwrite files
9. **edit_file** - Selective edits using pattern matching, whitespace normalization, dry-run preview
10. **move_file** - Rename or relocate files and directories

#### Search & Metadata
11. **search_files** - Recursively find files matching glob patterns with exclusion support (GITIGNORE-AWARE)
12. **get_file_info** - Retrieve detailed metadata (size, timestamps, permissions, type)
13. **list_allowed_directories** - Display accessible directories

### Key Features
- ✅ **Structured JSON output** - directory_tree returns clean JSON structure
- ✅ **Gitignore-aware** - search_files supports exclusion patterns
- ✅ **Security-first** - Path validation, restricted to allowed directories
- ✅ **Token-efficient** - Structured data, no parsing needed
- ✅ **Official support** - Maintained by Anthropic, will stay current
- ✅ **Widely adopted** - Thousands of users, battle-tested

### Installation

```bash
npx -y @modelcontextprotocol/server-filesystem /path/to/allowed/dir
```

## Recommendation

**DO NOT BUILD `idlergear fs`** - The official @modelcontextprotocol/server-filesystem ALREADY provides everything we need:
- Structured JSON outputs ✅
- Gitignore-aware search ✅
- Tree views ✅
- Token efficiency ✅
- Security ✅

### Instead, IdlerGear should:
1. **DEPEND** on @modelcontextprotocol/server-filesystem
2. **INTEGRATE** it as a default MCP server
3. **DOCUMENT** how to use it with IdlerGear
4. **EXTEND** only if there are IdlerGear-specific needs (e.g., linking filesystem operations to tasks/notes)

### Potential IdlerGear-Specific Extensions

If we find gaps later, we could add:
- **idlergear fs link** - Link file changes to tasks/notes
- **idlergear fs track** - Track which files are being worked on for a task
- **idlergear fs smart-ignore** - Project-type-aware ignore patterns (.idlergear additions to .gitignore)

But these would be THIN WRAPPERS around the official server, not reimplementations.
