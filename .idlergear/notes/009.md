# Research: Existing MCP Git Servers

## Executive Summary

**RECOMMENDATION: INTEGRATE, DON'T BUILD** - Multiple high-quality git MCP servers exist. Use official @modelcontextprotocol mcp-server-git for basic operations, integrate GitHub MCP for GitHub-specific features.

## Official Anthropic Git MCP Server

**Package:** `mcp-server-git` (Python, via `uvx`)
**Status:** Official reference implementation
**Repository:** https://github.com/modelcontextprotocol/servers/tree/main/src/git

### Tools Provided (14 total)

#### Status & Information
1. **git_status** - Working tree status
2. **git_log** - Commit history (supports date filtering, max_count)
3. **git_branch** - List branches (local/remote/all, filter by contains)

#### Diff Operations
4. **git_diff_unstaged** - Changes in working directory not staged
5. **git_diff_staged** - Changes staged for commit
6. **git_diff** - Differences between branches/commits

#### Staging & Committing
7. **git_add** - Add files to staging area
8. **git_reset** - Unstage all staged changes
9. **git_commit** - Record changes (returns commit hash)

#### Branch Management
10. **git_create_branch** - Create new branches
11. **git_checkout** - Switch branches

#### Commit Inspection
12. **git_show** - Show commit contents (hash/branch/tag)

### Key Features
- ✅ **Python-based** - Easy to integrate with IdlerGear
- ✅ **Simple, focused** - Core git operations only
- ✅ **Official support** - Maintained by Anthropic
- ✅ **Structured output** - Returns formatted text

### Configuration
```bash
uvx mcp-server-git --repository /path/to/repo
```

## Alternative: cyanheads/git-mcp-server (Feature-Rich)

**Package:** `@cyanheads/git-mcp-server` (Node.js)
**Status:** Community-maintained, production-grade
**Repository:** https://github.com/cyanheads/git-mcp-server

### Tools Provided (27 total)

#### Everything from Official + Advanced Features:
- **Repository**: init, clone, clean
- **Remote Operations**: fetch, pull, push, remote management
- **Advanced Workflows**: tag, stash, rebase, cherry-pick, worktree, reflog
- **History**: blame, reflog
- **Destructive Operations**: reset (with confirmations)
- **Commit Signing**: GPG/SSH support

### Advanced Features
- ✅ **Dual-output architecture** - JSON for LLM, Markdown for humans
- ✅ **Configurable verbosity** - minimal/standard/full
- ✅ **Cross-runtime** - Works with Node.js and Bun
- ✅ **Provider-based architecture** - Pluggable git implementation
- ✅ **Safety features** - Confirmations for destructive ops
- ✅ **Session management** - Per-session working directory context
- ✅ **Structured logging** - File rotation, console, MCP notifications

### Configuration
Environment variables:
- `MCP_RESPONSE_FORMAT` - json/markdown/auto
- `MCP_RESPONSE_VERBOSITY` - minimal/standard/full

## GitHub Official MCP Server

**Package:** `github-mcp-server`
**Status:** Official from GitHub
**Scope:** GitHub platform integration (NOT local git)

### Capabilities (GitHub Platform Only)
- Repository browsing, file search, commit analysis
- Issue & PR creation, updates, management
- GitHub Actions monitoring, workflow analysis
- Security alerts, Dependabot
- Team collaboration, discussions, notifications
- **Toolset system** - Fine-grained permission control

### Key Difference
- **Git MCP**: Local repository operations (commit, branch, diff)
- **GitHub MCP**: GitHub platform features (issues, PRs, Actions, security)

This is complementary, not a replacement for git operations.

## Recommendation

**DO NOT BUILD `idlergear git`** - Excellent servers already exist.

### Integration Strategy

**Option A: Dual Integration (Recommended)**
1. **Use cyanheads/git-mcp-server** as primary - Superior JSON output, verbosity control
2. **Use github-mcp-server** for GitHub features - Issues, PRs, Actions
3. **IdlerGear adds value** via thin wrappers:
   - `idlergear git commit-task <task_id>` - Auto-link commits to tasks
   - `idlergear git status --for-task <id>` - Filter status by task files
   - `idlergear git sync` - Update task status from commit messages

**Option B: Official + Extensions**
1. **Use mcp-server-git** (official) - Simpler, Python-based
2. **IdlerGear wraps** for structured output conversion
3. **Add IdlerGear-specific features** (task linking, etc.)

### IdlerGear-Specific Value Adds

If we wrap/extend, focus on:

1. **Task-Commit Linking**
   - Auto-add task references to commits: `feat: add feature (#42)`
   - Track which commits belong to which tasks
   - Auto-close tasks on merge

2. **Smart Status Filtering**
   - Show only files relevant to current task
   - Filter by IdlerGear project context
   - Exclude .idlergear/ files from noise

3. **Documentation Sync**
   - Detect when code changes require doc updates
   - Link commits to reference documents
   - Track when APIs change

4. **Knowledge Capture**
   - Auto-create notes from commit messages
   - Extract learnings from PR descriptions
   - Build knowledge graph from git history

## Recommended Implementation

**Phase 1: Integration (Quick Win)**
- Add cyanheads/git-mcp-server as recommended dependency
- Document configuration in .goosehints template
- Use it directly (no wrapper needed yet)

**Phase 2: IdlerGear Extensions (If Needed)**
Only build thin wrappers if we find gaps:
- `idlergear git task-commit` - Commit with automatic task linking
- `idlergear git task-status` - Status filtered by task context
- `idlergear git capture` - Extract knowledge from git history

**Phase 3: Advanced (Future)**
- Bi-directional hooks (git → IdlerGear, IdlerGear → git)
- Automatic knowledge extraction from commits
- Task state synchronization via commit messages
